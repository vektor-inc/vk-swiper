# CSS / SCSS コーディングルール

`.css` および `.scss` ファイルを編集する際に適用するルールです。

## 変数・ユーティリティ

- サイト全体で共通して使う色・フォント・余白などの値は、`variables.scss` などに CSS カスタムプロパティとして定義して一元管理してください。
- 特定のセレクタ内で CSS カスタムプロパティの使用を検討する場合、そのプロパティがサイト全体で共通利用できるものでないか先に検討してください。共通化できる場合はグローバルなカスタムプロパティとして定義し、それを割り当ててください。そのセレクタ内でしか使わない値であれば、数値を直書きしてかまいません。
- CSSカスタムプロパティの使用を優先し、SCSS変数は必要に迫られなければ使わないでください。
- テーマや見た目の一貫性を担保するため、既存の変数やユーティリティクラスを優先して使用してください。

## セレクター・クラス設計

- 共通のコンポーネントや変数を極力使用し、独自のクラス名に対する CSS は極力増やさないようにしてください。
- コンポーネント単位で責務を分け、影響範囲が明確になる構成にしてください。
- 詳細度（specificity）を必要以上に上げないようにしてください。`!important` は原則使用禁止です。

## SCSS 固有のルール

- ネストは深くなりすぎないようにしてください（目安：3階層以内）。
- `@mixin` / `@extend` は重複を減らすために活用しますが、乱用によって出力 CSS が肥大化しないよう注意してください。
