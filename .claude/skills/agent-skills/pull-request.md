# プルリクエストのルール

## タイトルと概要

- PR のタイトル・概要（description）は日本語で記載してください。
- タイトルは `[ 種類 ] 変更内容` の形式で記載してください。種類は以下から選択してください。
  - `[ 機能追加 ]`
  - `[ 不具合修正 ]`
  - `[ デザイン不具合修正 ]`
  - `[ セキュリティ修正 ]`
  - `[ 仕様変更 ]`
  - `[ 開発環境 ]`
  - `[ その他 ]`
- 変更内容の部分は、一般ユーザーが読んでも何が変わったか分かるような表現にしてください。
- 変更対象が特定の WordPress ブロックの場合は、種類の後にブロック名を `[ ブロック名 ]` の形式で記載してください。対象ブロックが Pro 版のみで提供されているものの場合は、ブロック名の後に `(Pro)` を付けてください。ただし、ブロック名自体に既に「Pro」が含まれている場合は `(Pro)` を追加しないでください。
- 例：`[ 機能追加 ] 投稿一覧にカテゴリーフィルターを追加`、`[ 不具合修正 ] モバイル表示でメニューが崩れる問題を修正`、`[ 不具合修正 ][ 投稿リストブロック (Pro) ] 投稿の取得件数が0になってしまう不具合を修正`

## テスト

- PR では手動テストの内容や確認観点を記載します。

## CodeRabbit のレビュー対応

PR作成後、以下の手順でCodeRabbitのレビューを監視してください：

1. 監視開始時刻を `date +%s` で記録する
2. 30秒待機（`sleep 30`）してから以下の2エンドポイントをそれぞれ呼び出し、結果を合算する
   - `gh api repos/{owner}/{repo}/issues/{PR番号}/comments`（PRへの通常コメント）
   - `gh api repos/{owner}/{repo}/pulls/{PR番号}/comments`（インラインレビューコメント）
3. 合算した全コメントの中で `coderabbitai` によるコメントに「Prompt for AI Agents」が含まれているか確認する
   - **含まれている場合**: 指摘内容を確認する
     - PR趣旨・スコープに沿った指摘であれば修正してpushする
     - 指摘は正当だがこのPRのスコープ外であれば、該当コメントに返信して `@coderabbitai` に別issueとして立てるよう依頼する（例: `@coderabbitai この指摘はこのPRのスコープ外のため、別issueとして登録をお願いします。`）
     - その後ステップ2に戻る
   - **含まれていない場合**: ステップ4へ
4. 現在時刻を `date +%s` で取得し、監視開始から300秒（5分）経過しているか確認する
   - **経過していない場合**: ステップ2に戻る
   - **経過した場合**: 「CodeRabbitのレビューコメントは確認されませんでした」と報告して終了する
